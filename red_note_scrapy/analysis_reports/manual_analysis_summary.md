# 小红书子评论API手动分析总结

## 📋 分析概况

**分析目标**: 小红书子评论API逆向工程

## 🎯 已确认的关键发现

### 1. API结构分析 ✅
- **正确路径**: `/api/sns/web/v2/comment/sub/page`
- **正确参数名**: `root_comment_id` (不是`comment_id`)
- **必需参数**: 
  - `note_id`: 笔记ID
  - `root_comment_id`: 根评论ID
  - `num`: 返回数量
  - `image_formats`: 图片格式
  - `xsec_token`: 安全令牌
  - `cursor`: 分页游标

### 2. 认证机制分析 ✅
- **关键发现**: `x-s-common = X-s` (两者必须相同)
- **Cookie结构**: 12个组件，包含认证、安全、会话信息
- **请求头要求**: 完整的浏览器头信息
- **时间戳**: 动态生成，需要更新loadts

### 3. X-s参数特征分析 ✅
- **格式**: `XYS_` + Base64编码内容
- **长度**: 固定328字符
- **前缀**: `XYS_` (XiaoHongShu Sign)
- **编码**: Base64，解码后241字节
- **字节分布**: 52.7%扩展ASCII，34.0%可打印ASCII，13.3%控制字符

## 🔍 深入分析成果

### 1. Cookie组件详解
```
认证组件:
- a1: 用户认证令牌 (52字符)
- web_session: Web会话ID (38字符)  
- webId: 浏览器指纹ID (32字符)

安全组件:
- websectiga: 安全验证参数 (64字符)
- sec_poison_id: 安全防护ID (36字符)
- acw_tc: 验证码相关 (62字符)

会话组件:
- gid: 访客ID (72字符)
- xsecappid: 安全应用ID (10字符)
- abRequestId: A/B测试请求ID (36字符)

配置组件:
- webBuild: 前端版本号 (6字符)
- unread: 未读消息状态 (URL编码JSON)
- loadts: 加载时间戳 (13字符)
```

### 2. X-s生成算法尝试
✅ **已排除的简单算法**:
- 纯时间戳
- URL哈希 (MD5/SHA1/SHA256/SHA512)
- 时间戳+URL组合
- 用户ID+时间戳
- 复合参数哈希
- HMAC签名 (多种密钥)

✅ **已确认的特征**:
- 非标准加密算法
- 涉及多个参数组合
- 可能使用自定义密钥/盐值
- 与前端JS代码紧密相关
